Descrizione

Protoweb è un'applicazione Blazor (serverside) che consente di acquisire da file system files immagini in tutte le estensioni comuni e messaggi di outlook con estensione msg tramite drag & drop degli allegati o dell'intero messaggio di posta.

Installazione

Premessa

Le pagine del sito che contiene l'app vengono raggiunte da sharepoint tramite il progetto SharePointBlazorPages che risiede su server Sharepoint e consente di accedere all'applicazione previa autenticazione dell'utente su sharepoint e rilascio del ticket. Per informazioni su SharepointBlazorPages e sul sistema di rilascio dei Ticket gestiti dal Servizio security tickets potete consultare la documentazione presente nel vostro wiki

Le SharePointBlazorPages che interessano Protoweb sono le seguenti:

Front End

  • La pagina di front end che consentira di gestire gli allegati da protocollare è protocollo.aspx

Back End

  • La pagina che servere a difinire gli attributi del protocollo è protocolloattributi
  • La Pagina che server a gestire le categorie di protocollo è protocollocategorie
  • La pagina di gestione dei lock utile a rimuovore lock attivi è tagframeworklocks

Risorse applicazione protoweb

Risorsa Server Site Porta
Protocollo. Sito di front end realizzato in Blazor SRV026 Protocollo 8009
TagPortalBackEnd. Sito di Back End reaslizzato in .net 5.0 Web Api REST per gestire le transazioni e la persistenza dei dati SRV026 TagPortalBackEnd 8008

Invio su mago

Per le fatture / notecredito CEE ed EXTRA CEE che arrivano via mail, la porocedura di protocollo (tramite la creazione di un'apposita categoria) consente di inviare (al momento del salvataggio) alla tabella di mago "MA_EI_ITImportDocument" i dati del documento.

Per consentire un corretto invio sono stati creati degli enumerativi che consentono di associare agli attributi definiti in fase di input la tipologia di attributo. Ciò consentirà al backend di inviare correttamente i dati.

public class Enumerativi
    {
        /// <summary>
        /// enumerativo per indicare il tipo di attributo nel caso di un invio su mago
        /// </summary>
        public enum AttributiFatturazionePassiva 
        {
            Nessuno,
            NrDoc,
            DataDoc,
            Fornitore,
            TipoSdi,
            ImportoTotale,
            DataRicezione,
            ModelloContabile
        }

}

Un esempio di categoria protocollo in cui sono definiti gli enumerativi per un invio su mago. NB la possibilità di definire i tipi attributo mago è possibile solo se per la categoria il flag "Invio su mago è true"

Impostazioni Modello contabile.

Per conformità di comportamento della procedura easybuilder di fatturazione passiva il modello contabile con cui dovrà essere generato il documento su mago deve essere impostato come un nodo dell'xml che contiene i dati della fattura elettronica.

Il protocollo invia uno schema xml fittizio (in quanto si tratta di fatture non provenienti dallo SDI) e in base al tipo di fornitore presente valorizzerà i seguenti nodi dello schema con i seguenti valori

Nodo xml Natura Descrizione
P1.0 Fornitore CEE
P1.1 Fornitore EXTRACEE
